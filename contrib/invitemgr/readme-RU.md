# Matrix Invite Manager Widget

Микросервис и кастомный виджет для Matrix Synapse, который позволяет пользователям сервера самостоятельно создавать одноразовые пригласительные ссылки (инвайты) для регистрации новых участников без участия администратора. Использует Matrix Registration Tokens.

## Основные возможности

*   **Интеграция с Element:** Работает как кастомный виджет внутри Matrix-клиента.
*   **Управление инвайтами:** Удобный интерфейс для создания и удаления пригласительных ссылок.
*   **Лимиты:** Настраиваемое ограничение количества активных токенов на одного пользователя.
*   **Безопасность:** 
    *   Аутентификация пользователей через Matrix OpenID Connect (OIDC).
    *   Автоматическое обновление токена администратора Synapse.
    *   Защита публичного эндпоинта от перебора токенов (Rate limiting и Tarpitting).
*   **Страница гостя:** Лендинг для приглашенных пользователей с проверкой валидности пригласительного токена и автоматическим переходом к регистрации.
*   **Мультиязычность:** Полная поддержка русского и английского языков (i18n). Автоматические применение языка в зависимости от локали браузера. Вы можете легко добавить свой язык при помощи JSON файлов.
*   **Возможность установки в закрытом сегменте** - По умолчанию виджет использует локальную копию matrix-widget-api, базирующуюся на версии 1.17.0. При необходимости Вы можете переключиться на online версию.

## Архитектура

1.  **Backend (Go):** Взаимодействует с Synapse Admin API, управляет базой данных SQLite и проверяет права доступа.
2.  **Private UI (HTML/JS):** Панель управления для участников сервера, открываемая внутри комнаты Matrix.
3.  **Guest UI (HTML/JS):** Страница приветствия, на которую попадает гость по ссылке вида `https://domain.com/invites/{token}`.
4.  **Index UI (HTML):** Страница-заглушка для домена микросервиса, *для защиты от особо любопытных =)*.

## Требования

*   Установленный Matrix Synapse с включенными [регистрационными токенами](https://element-hq.github.io/synapse/latest/usage/configuration/config_documentation.html#registration_requires_token).
*   Учетная запись администратора Synapse.
*   Установленный docker и docker compose
*   Настроенный обратный прокси (Nginx, Caddy и др.) для работы по HTTPS.

## Настройка

Сервис настраивается через переменные окружения:

| Переменная | Описание | Пример использования| Обязательно | Значение по умолчанию |
| :--- | :--- | :--- | :--- | :--- |
| `SYNAPSE_URL` | URL вашего сервера synapse | `https://matrix.domain.com` | ✅ | - |
| `SYNAPSE_ADMIN_USER` | Имя пользователя администратора | `admin` | ✅ | - |
| `SYNAPSE_ADMIN_PASSWORD` | Пароль администратора | `your-password` | ✅ | - |
| `WEBCLIENT_URL` | URL веб-клиента Element (на него будет происходить перенаправление) | `https://chat.domain.com` | ✅ |
| `PORT` | Порт внутреннего сервера (бэкэнда) | `8080` | ❌ | `8080` |
| `PROXY_ADDR` | IP/CIDR прокси-сервера | `172.18.0.0/16` | ❌ | - |
| `INVITES_PER_USER` | Макс. кол-во активных инвайтов на юзера | `3` | ❌ | `5` |
| `WIDGET_API` | Режим JS-API (`internal` или `external`) | `internal` | ❌ | `internal` | 
| `WIDGET_API_URL` | URL для подключениея внешнего JS-API. Используется только при `WIDGET_API = external`   | `https://esm.sh/matrix-widget-api@latest` | ❌ | `https://esm.sh/matrix-widget-api@1.17.0` |  
| `NO_TLS` | **Не рекомендуется** Отключение проверки HTTPS. Укажите значение `I_KNOW_WHAT_I_DO` для включения опции   | - | ❌ | - |

## Установка и запуск

### Docker

0. *Настоятельно рекомендуем создать отдельного пользователя-администратора для данного микросервиса.*

1. Скопируйте содержимое репозитория на свой сервер (для примера я буду использовать директорию ./plugins/invitemgr)
2. Настоятельно рекомендуется запускать сервис в Docker-контейнере, позади проксификатора (например Caddy):

```Caddyfile
# Сервис инвайтов
invites.domen.ru {
    reverse_proxy invitemgr:8080
}
```

3. Настройте Ваш docker-compose файл:
```docker-compose.yaml
services:
  invitemgr:
    build: ./plugins/invitemgr
    restart: unless-stopped
    environment:
      - SYNAPSE_URL=https://matrix.domain.org
      - SYNAPSE_ADMIN_USER=admin_bot_user
      - SYNAPSE_ADMIN_PASSWORD=VERY_STRONG_PASSWORD
      - WEBCLIENT_URL=https://chat.domain.org
      - PROXY_ADDR=172.18.0.0/16 #Docker subnet
      - WIDGET_API=internal
      - INVITES_PER_USER=10
    volumes:
      - ./plugins/invitemgr/db:/app/data
      - ./plugins/invitemgr/libs:/app/libs
    networks:
      - your-matrix-network
```

4. Сделайте build образа и запустите его:
```bash
docker compose build invitemgr
docker compose up -d invitemgr
```
5. Посмотрите лог образа, убедитесь в отсутствии ошибок. При необходимости - поправьте конфиг.
```bash
docker compose logs invitemgr
```

## Использование

1. Зайдите в комнату внутри вашего Element, где Вы хотите разместить виджет.
2. Введите команду для подключения виджета внутри комнаты:
```
/addwidget https://invites.domain.org
```
3. Разрешите виджету доступ к вашим данным (OpenID). *Это будет необходимо сделать каждому пользователю, который захочет работать с виджетом/приглашать других пользователей*.
    3.1. Проверка - сгенерируйте инвайт.
    3.2. Зайдите в вашу панель администратора synapse, проверьте в разделе "Токены регистрации" что токен действительно создался и он соответствует сгенерированному инвайту.
    3.3. Перейдите по сгенерированной ссылке и попробуйте завершить регистрацию.
4. При необходимости, закрепите виджет в шаблоне комнаты.

**Теперь любой пользователь, который имеет доступ в настроенную комнату, может создавать инвайты и приглашать других пользователей.**
